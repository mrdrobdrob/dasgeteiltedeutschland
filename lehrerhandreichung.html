📋 Lehrerhandreichung
Operation Mauerfall – Escape Room | Klasse 10 | Geschichte

Auf einen Blick: Interaktiver digitaler Escape Room zum Thema "Das geteilte Deutschland" (1945–1989). 10 Stationen, 135 Minuten (3 Schulstunden), Gruppenarbeit (4–5 SuS pro Gruppe), Schulbuch Cornelsen S. 88–123.

1. Vorbereitung & Setup

A) GitHub Pages einrichten (einmalig, ~10 Minuten)

  1. GitHub-Account anlegen (falls noch nicht vorhanden): github.com
  2. Neues Repository erstellen:
    • Klickt auf das +-Symbol oben rechts → "New repository"
    • Name: z.B. escape-room-mauerfall
    • Sichtbarkeit: Public (wichtig für GitHub Pages)
    • "Create repository" klicken
  3. Dateien hochladen:
    • Klickt auf "Add file" → "Upload files"
    • Alle Dateien aus dem ZIP-Ordner per Drag & Drop hochladen:
      index.html, station01.html bis station10.html, success.html, shared.css, utils.js
    • "Commit changes" klicken
  4. GitHub Pages aktivieren:
    • Im Repository → "Settings" (oben)
    • Linkes Menü: "Pages"
    • Source: "Deploy from a branch"
    • Branch: main / root → "Save"
    • Nach 1–2 Minuten erscheint die URL: https://[username].github.io/[repo-name]/
  5. URL testen & teilen: Öffnet die URL im Browser. Wenn die Startseite erscheint, ist alles fertig!
⚠️ Wichtig: Das Repository muss Public sein, damit GitHub Pages kostenlos funktioniert. Es werden keine persönlichen Schülerdaten auf GitHub gespeichert – alle Daten liegen nur im Browser-Speicher (localStorage) des Schülergeräts.

B) Technische Voraussetzungen (Klassenzimmer)

  • 1 Tablet oder Laptop pro Gruppe (4–5 SuS)
  • Internetverbindung (nur für das Laden der Google-Fonts – funktioniert auch offline, dann mit Fallback-Schrift)
  • Jede Gruppe kennt die URL der Startseite
  • Schulbücher: Cornelsen, Kapitel 3 (S. 88–123) – ein Exemplar pro Gruppe mindestens
  • Hefte für alle SuS (Agenten-Akte)

C) Vorbereitung im Unterricht (Tag vorher)

  • URL auf Tafel/Beamer/WhatsApp-Gruppe bereitstellen
  • Ggf. kurze Einführung: "Morgen machen wir einen Escape Room. Ihr braucht eure Schulbücher und Hefte."
  • Gruppen einteilen (4–5 SuS, möglichst leistungsheterogen)

2. Detaillierter Zeitplan (135 Minuten)

0:00 – 0:10
Stunde 1 Einführung & Einstieg
Szenario vorlesen (oder Startseite zeigen), Gruppen bilden, URL bekanntgeben. Kurze Erklärung: Schulbuch als Hauptquelle, Heft für jede Station nutzen.
0:10 – 0:45

0:50 – 1:30

Kernstationen mit viel Inhalt. Hier entscheidet sich der Rhythmus der Gruppen.
1:30 – 1:35
Zwischenfazit: Welche Gruppe hat Station 7 schon? Kurze Frage: "Was war das Wirtschaftswunder?" – 1 Gruppe antwortet.
1:35 – 2:05

Finale Stationen. Erste Gruppen, die fertig sind, beginnen mit der Reflexionsaufgabe.
2:05 – 2:20

2:20 – 2:30
Reflexion & Abschluss: Was hat überrascht? Was fandet ihr schwierig? Welche Station war lehrreichst?
2:30 – 2:35
Optional Gruppen, die fertig sind, drucken ihr Zertifikat aus (Erfolgsmeldung → Drucken).

4. Differenzierung & Umgang mit heterogenen Gruppen

Für schnelle Gruppen:

  • Die Heft-Aufgaben bieten Vertiefung – ermutigt sie, diese besonders ausführlich zu bearbeiten
  • Zusatzfrage: "Wenn ihr Bundeskanzler wärt: Welche Maßnahme aus diesem Escape Room hättet ihr anders gemacht?"
  • Recherche-Erweiterung: Stolpersteine-Projekt (Station 4-Hinweis im Schulbuch)

Für langsame Gruppen:

  • Jede Station hat einen eingebauten Hinweis-Button (💡) – erinnert Gruppen daran, wenn sie 5+ Minuten feststecken
  • Stationen können übersprungen werden, wenn nötig (direkter URL-Aufruf der nächsten Station)
  • Alternativ: Station 7 (Wirtschaft) ist einsteigerfreundlicher und kann als "Booster" dienen

Notfall-Codes:

Wenn eine Gruppe komplett feststeckt und keine Zeit mehr ist, können Sie folgende URLs direkt aufrufen:

7. Datenschutz & Technik

  • Keine Daten auf Servern: Alle Eingaben (Teamname, Fortschritt) werden nur im localStorage des Browsers gespeichert und beim nächsten Browser-Reset gelöscht.
  • Keine Registrierung: SuS müssen keinen Account erstellen.
  • Kein Tracking: Keine Analyse-Tools oder Cookies.
  • Offline-Fähigkeit: Nach dem ersten Laden funktioniert der Escape Room auch ohne stabiles Internet (außer Google Fonts – Fallback auf Systemschrift).
